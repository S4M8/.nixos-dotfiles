# .nixos-dotfiles

Declarative NixOS configuration, home-manager modules, and managed dotfiles for a multi-machine environment.

## Architecture

### Machine inventory

| Host | Role | Hardware |
|------|------|----------|
| `hexbox` | Laptop (daily driver) | AMD Barcelo (integrated GPU) |
| `box` | Workstation (in-progress) | Dedicated NVIDIA GPU, secondary storage |

### Repository layout

```
~/.nixos-dotfiles/
├── flake.nix               # NixOS configurations: hexbox + box
├── flake.lock
├── configuration.nix       # Shared system module imports
├── home.nix                # Shared home-manager configuration
├── hosts/
│   ├── hexbox/             # Laptop: hostname, hardware, modules
│   └── box/                # Workstation: hostname, hardware, modules
├── modules/
│   ├── system/             # Shared NixOS modules (services, hardware, display, etc.)
│   └── home/               # Shared home-manager modules (shell, dev, media, etc.)
├── config/                 # Dotfiles (symlinked via home-manager)
│   ├── hypr/               #   Hyprland window manager
│   ├── quickshell/         #   QML desktop shell
│   ├── nvim/               #   Neovim editor
│   ├── kitty/              #   Kitty terminal
│   └── ...                 #   Additional managed configs
├── secrets/                # Agenix-encrypted secrets
│   ├── pubkeys/            #   Known machine public keys
│   ├── rekey.sh            #   Re-encryption utility
│   └── *.age               #   Encrypted secret payloads
├── bootstrap.sh            # New machine provisioning
├── validate.sh             # Pre-rebuild validation
├── .gitignore
└── README.md
```

## Getting started

### Initial setup on a new machine

```bash
git clone git@github.com:S4M8/.nixos-dotfiles.git ~/.nixos-dotfiles
cd ~/.nixos-dotfiles
./bootstrap.sh
```

The bootstrap script walks through:

1. **Identity** — hostname, timezone
2. **Hardware detection** — runs `nixos-generate-config`
3. **Wallpapers** — clones the wallpapers repository to `~/pictures/wallpapers/`
4. **Age key** — generates a machine-local age keypair for secret decryption
5. **Host configuration** — creates `hosts/<hostname>/base.nix`, `modules.nix`, `home.nix`
6. **System symlink** — links `/etc/nixos` → `~/.nixos-dotfiles`
7. **Rebuild** — runs `nixos-rebuild switch`

### Manual setup

```bash
sudo rm -rf /etc/nixos
sudo ln -s ~/.nixos-dotfiles /etc/nixos
sudo nixos-rebuild switch
```

## Module architecture

Shared and per-host module loading follows a layered approach:

- `configuration.nix` — imports all modules common to every machine (boot, display, networking, services, fonts, security, users, packages, hardware defaults)
- `hosts/<host>/modules.nix` — imports only host-specific modules (GPU driver, additional filesystems, service overrides)
- `hosts/<host>/base.nix` — hostname and timezone assignment

This avoids duplicating the common module import list across per-host files.

## Secret management

API tokens (TMDB, AniList) are encrypted and versioned in-repo using [agenix](https://github.com/ryantm/agenix). Decryption occurs at build time via a machine-local age private key.

### Key material locations

| Path | Role |
|---|---|
| `~/.config/age/key.txt` | User age private key (generated by bootstrap) |
| `/etc/age/keys.txt` | System age private key (copied on first rebuild) |
| `secrets/*.age` | Encrypted secrets (committed to repository) |
| `secrets/pubkeys/*.age` | Plaintext public keys for each known machine |

### Onboarding a new machine

When bootstrap.sh runs on a new machine, it generates a fresh age keypair and prints the public key. To authorize the new machine for secret decryption:

```
# On a machine already in the repository (e.g., hexbox):
echo '<public-key-from-new-machine>' > secrets/pubkeys/<hostname>.age

# Re-encrypt all secrets to include the new key:
bash secrets/rekey.sh

# Commit the updated secrets:
git add secrets/
git commit -m "secrets: add <hostname> public key"
git push

# On the new machine, pull and rebuild:
cd ~/.nixos-dotfiles && git pull
sudo nixos-rebuild switch
```

The `rekey.sh` script decrypts each `.age` file using the local age identity, then re-encrypts it for every public key in `secrets/pubkeys/`. All encrypted files are updated in-place.

### Creating a new secret

```bash
# Encrypt for one machine, then rekey for all:
echo '<token-value>' > /tmp/plain
age -r "$(cat secrets/pubkeys/hexbox.age)" -o secrets/<name>.age /tmp/plain
rm /tmp/plain

# Re-encrypt for all registered machines:
bash secrets/rekey.sh

git add secrets/
git commit -m "secrets: add <name>"
```

## Wallpapers

The wallpaper collection is maintained in a [separate repository](https://github.com/S4M8/wallpapers). It is cloned to `~/pictures/wallpapers/` during bootstrap. The QuickShell wallpaper picker and pywal-based theming system reference this path.

## Validation

Before rebuilding, run:

```bash
./validate.sh [hostname]
```

This checks:
- NixOS version compatibility with the flake
- `/etc/nixos` symlink integrity
- Per-host configuration file existence
- Required secrets and credentials
- Flake evaluation via `nix flake check`
